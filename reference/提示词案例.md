请你为我生成一个可安装、可复用的 Codex Skill，Skill 名称为：

creator-content-analysis-skill

目标：
这个 Skill 用于分析某个自媒体博主/达人/KOL/账号的所有已发布内容，自动总结该账号的内容调性、内容风格、赛道定位、赛道打法、内容标签、视频结构、爆款规律、商业化潜力，并输出结构化分析报告。

一、Skill 的核心使用场景

用户可以输入以下任意一种数据：

1. 自媒体博主主页链接，例如抖音、小红书、B站、视频号、微博、TikTok、YouTube、Instagram 等；
2. 单条或多条视频/图文链接；
3. 已整理好的 CSV / JSON / Markdown / TXT / Excel 文件；
4. 视频标题列表、文案列表、转写稿列表、评论列表、点赞收藏转发数据；
5. 用户手动粘贴的博主内容样本。

Skill 需要根据输入内容，自动生成账号分析报告。

二、Skill 需要分析的维度

请至少包含以下分析模块：

1. 账号基础画像分析

* 账号名称
* 平台
* 主页链接
* 粉丝量
* 内容数量
* 主要内容类型
* 目标用户人群
* 账号定位一句话总结

2. 内容赛道分析

* 一级赛道，例如科技、AI、职场、财经、情感、教育、本地生活、美妆、母婴、汽车、房产、旅游、知识科普、娱乐、短剧、影视解说、带货种草等
* 二级细分赛道
* 赛道竞争强度
* 赛道商业化潜力
* 赛道用户需求
* 该账号在赛道中的差异化定位

3. 内容调性分析
   请判断账号整体调性，例如：

* 专业理性型
* 情绪共鸣型
* 犀利观点型
* 幽默搞笑型
* 干货教学型
* 个人陪伴型
* 反差人设型
* 强销售转化型
* 故事叙事型
* 观点输出型
* 热点借势型

每种调性都要给出判断依据。

4. 内容风格分析
   分析以下内容：

* 标题风格
* 封面风格
* 开头 3 秒钩子
* 叙事结构
* 语言表达习惯
* 镜头语言
* 剪辑节奏
* 字幕风格
* BGM/音效使用
* 结尾引导方式
* 评论区互动方式
* 是否具有强人设
* 是否适合矩阵化复制

5. 单条内容结构分析
   对每条内容输出：

* 标题
* 发布时间
* 链接
* 核心主题
* 内容摘要
* 开头钩子
* 用户痛点
* 核心观点
* 情绪触发点
* 内容结构
* 转化点
* 爆点判断
* 可复刻元素
* 不足与风险

6. 内容标签体系
   为账号和单条内容自动打标签，包括：

* 赛道标签
* 人群标签
* 主题标签
* 情绪标签
* 表达风格标签
* 商业化标签
* 热点标签
* 内容形式标签
* 平台风格标签
* 传播潜力标签

7. 爆款规律分析
   分析账号内容中表现较好的内容，提炼：

* 高频主题
* 高频标题句式
* 高频开头方式
* 高频情绪触发点
* 高频选题方向
* 高频人群痛点
* 高频转化路径
* 爆款内容共性
* 非爆款内容问题

8. 商业化潜力分析
   分析：

* 适合广告合作的品类
* 适合带货的品类
* 适合知识付费的方向
* 适合私域转化的方向
* 适合品牌合作的方向
* 内容转化链路建议
* 账号商业化风险

9. 对标与复刻建议
   输出：

* 这个账号最值得学习的地方
* 哪些内容结构可以复刻
* 哪些标题模板可以复用
* 哪些选题可以继续扩展
* 如果我是同赛道新账号，应该如何模仿但不抄袭
* 给出 10 个可复刻选题
* 给出 10 个标题模板
* 给出 5 条账号优化建议

四、实现要求

1. 不要求直接破解或爬取平台接口，不要编写违反平台规则的爬虫。
2. 支持用户手动提供数据，也支持未来扩展合法 API 或浏览器采集结果。
3. 所有输入数据要统一转换成标准结构。
4. 输出结果要同时支持 Markdown 报告和 JSON 结构化结果。
5. 分析逻辑要尽量模块化，方便后续接入大模型 API。
6. 代码使用 Python 实现，要求简单、清晰、适合小白修改。
7. README 要写清楚安装方式、使用方式、输入格式、输出格式、扩展方式。
8. SKILL.md 要按照 Codex/Claude Code Skill 的写法，说明这个 Skill 什么时候触发、如何使用、输入输出规范、注意事项。
9. skill.json 要包含名称、描述、入口说明、适用场景。
10. 所有 prompt 文件要写得专业、可直接用于大模型分析。
