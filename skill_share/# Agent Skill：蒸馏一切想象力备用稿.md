以后 Agent 遇到同类需求，直接调用这个 Skill，不用从零重新设计步骤，复用这套现成的工作逻辑去完成任务


大家在使用codex、claude code、哪怕是豆包、deepseek的时候总会在不经意间重复询问类似的问题，这在同一个岗位从事经常性工作的时候比较明显，

第3页
Skill 就是给 AI 智能体 Agent 打造的成套能力工具箱，它把专业知识、固定办事流程与配套参考资料打包成可复用、易管理的独立能力单元；如果不使用 Skill，每次安排同类工作都要反复交代需求、重传资料，产出结果参差不齐很不稳定，而用上 Skill 后可以沉淀企业经验、重复复用高频工作流程，采用渐进式按需加载内容减少冗余指令，既能不用反复给 AI 写大量提示词、大幅减少沟通成本，也能让 AI 输出结果标准统一更稳定，还能像搭积木一样灵活给 Agent 拓展新业务能力，上下文精简克制，整体工作效率与交付质量都得到明显提升。

第4页
对于codex、claude code或者其他agent
Skill 就像是为 Agent 量身打造的各类专业武器包，会把任务上下文、行业专业知识、标准化做事步骤、可重复执行的操作统一封装起来；我们设计 Skill 的思路不能上来就盲目编写功能，而是要先判断不同用途的 Agent 具体欠缺什么专业能力，再针对性封装对应的 Skill，比如编码 Agent 配套代码规范、调试流程类技能，研究 Agent 配套检索论文分析类技能，营销、金融 Agent 分别匹配各自行业模板、指标、风控合规类技能；借助 Skill 补齐能力后，既能突破大模型原生能力的局限，让不同类型 Agent 稳定处理对应专业工作、每次输出结果标准统一，也不用反复给 AI 重复描述任务要求，大幅提升任务执行效率。


第5页
我们常常听到一些概念比如Prompt、MCP、Skill、Subagent 四个组件
我们先来厘清 Prompt、MCP、Skill、Subagent 四个组件各自定位与分工边界，说明它们如何配合搭建完整智能工作流：
Prompt 就是单次临时的口头指令，用来当场说清楚这一次任务的要求、格式与目标，只适合一次性需求，很难反复复用；
MCP 相当于外接插座与接口，专门帮 Agent 打通外部数据库、各类 API 与线上服务，获取外部数据；
Skill 是打包好可反复调用的能力手册，把成熟业务流程、脚本、模板、资料封装成稳定可移植的能力包，专门应对重复、多步骤的常态化工作；
Subagent 如同分工不同的专项小队，能拆分复杂大任务、隔离上下文并行干活，最后汇总结果；整体协作逻辑是用户先用 Prompt 传递需求意图，由主 Agent 判断调度，按需调用 MCP 获取外部数据、启用 Skill 执行标准化流程、派发 Subagent 拆分并行处理子任务，区别就是 Prompt 只管单次要什么，Skill 沉淀长期反复怎么做，各司其职搭配起来才能做出稳健复杂的智能工作流程。

第7页
一个Skill整体是独立文件夹结构，**SKILL.md是唯一必不可少的核心文件**，构成Skill最小运行闭环
文件顶部YAML格式的元信息用来定义名称、标签、版本等配置，决定这个Skill在什么场景下触发加载；下方Markdown正文写明任务目标、分步操作流程、输出格式，规定Skill被调用之后具体怎么执行；
除此之外还可按需选配四个文件夹：
scripts存放运行脚本、references存放参考资料、assets存放图片模板等资源、examples存放使用案例，
编写Skill的核心思路不是堆砌文字资料，而是沉淀一套完整可落地的执行流程，简单需求仅靠SKILL.md即可完成，复杂业务再搭配配套文件夹，实现内容按需加载、灵活扩展。
 
第8页
我们拿HR招聘筛简历这个真实场景，给大家演示一个完整Skill该怎么落地：HR日常要反复处理一堆PDF简历，对照销售、开发等固定岗位要求挨个筛选候选人，整理对比分析报告，还得把结果发邮件汇报领导，整套工作重复度特别高，刚好适合封装成Skill。这个Skill的输入就是批量简历文件，内置好各岗位招聘评判标准，最终输出规整的筛选报告，还支持一键发邮件；它和临时写一句提示词完全不一样，是一整套可以反复调用的人事工作流程，后续再招同类型岗位，只要丢进新一批简历、选定对应岗位方向就能自动跑完筛选，不用每次重新写一遍操作步骤，既能每批次稳定运行、岗位筛选标准统一复用，产出的报告也格式规整、可以直接上交使用。

第9页
让AI自动生成对应的Skill，核心原则就是：写提示词时必须把整套工作从头到尾交代完整，清晰写明输入来源、判断规则、最终产出、额外动作边界。完整需求逻辑是：AI先读取指定文件夹里的多份PDF简历，对照销售、开发两类岗位招聘标准逐一比对筛选，再参照本地固定模板生成标准化筛选报告；同时划定边界，只有用户明确提出要求时，才自动把报告发邮件，不默认主动发邮件，并且规定Skill内部所有说明文档、代码注释都统一用简体中文。整个流程拆解下来就是定位简历文件夹、解析PDF内容、匹配招聘门槛、按模板出报告、按需决定是否发邮件，也给我们定下写Skill提示词的规范：不能只说半截需求，必须把输入是什么、判断逻辑、输出样式、附带操作的约束条件全部写清楚，像发邮件这种有实际影响的动作，要把触发规则固化进Skill流程里，避免AI临场随意发挥、行为不可控。

第10页
这个页面拆解了简历筛选Skill完整的文件夹架构，整个Skill文件夹命名为resume-screener，SKILL.md是整个能力包的核心主体文件，文件内分为YAML格式元信息与Markdown正文指令两部分，元信息用来定义Skill名称、功能说明，判定触发使用的场景，正文写明任务目标、所需素材、完整执行步骤与最终输出形式；配套还有三个附属文件夹各司其职，scripts文件夹存放发邮件这类可执行脚本，references文件夹存放销售、开发两类岗位招聘评判标准文档，assets文件夹存放报告模板等样式资源。这套Skill还支持跨Skill联动调用，提取PDF文字时可以复用现成的PDF解析Skill，既减少重复开发、节省算力开销，单人使用可直接调用已有能力，多人协作场景则适合用脚本封装核心逻辑，提升整体独立性和可移植性；整体完整运行链路就是读取本地简历、调用PDF能力提取文字、对照招聘标准筛选人员、套用模板生成筛选报告，最后按需调用邮件脚本推送结果，每个文件和文件夹分工清晰，整套流程规整易维护。
 
第11页
我们发出筛选指定路径简历、并把结果邮件上报的指令之后，Claude Code并不会一股脑把所有技能的全部内容都塞进上下文，而是依靠这套渐进式披露机制分步加载内容。系统最先只会读取简历筛选Skill的名称、简介这类轻量化元信息，快速判断这个技能是否适配当下任务，确定需要启用之后，才接着载入SKILL.md里完整的执行流程，再按需调用解析PDF的附属Skill、发邮件脚本、招聘规范文档、报告模板等配套材料，一步步完成整套工作。平台内部还同时存放着发票自动处理、PPT处理等多个独立Skill，每一个Skill都有专属隔离的上下文空间，只有任务匹配触发对应技能时，它内部的各类资料才会逐步释放读取，其余暂时用不到的技能全程不会参与加载，既能避免上下文内容杂乱冗余、不同业务信息互相干扰，也能有效节约模型算力开销，就算多个技能并存运行，也能保证每一项任务都跑得高效、稳定。

## 第一张PPT（项目简介 dot-skill）
我来给大家介绍dot-skill项目，通俗来讲这个项目的核心能力，就是把某一个人的说话习惯、做事思路完整提取出来，打包成一个专属AI技能。我们先收集聊天记录、往来邮件、工作文档、视频字幕这些原始资料，再补充上这个人的性格特点、个人背景、这个技能的使用目标，系统就会自动提炼、蒸馏生成专属Skill；启用之后AI说话、思考模式都会无限贴近本人，真正做到像这个人一样沟通办事。这个产品是从最早只针对职场同事的colleague-skill升级而来，现在适用范围特别广，不管是同事、家人伴侣、朋友偶像、知名人物、小说角色，甚至我们自己本人，都能被蒸馏成独立Skill，这套能力不光复刻说话语气，还能完整保留对方处理问题的逻辑、沟通上下文、决策框架，最终的目的，就是让AI不再是千篇一律的模板化回答，而是能复刻特定人的思维模式，变得更有真人特质。

## 第二张PPT（大版本更新内容）
接下来我们聊聊这次版本迭代带来的变化，产品从初代只能做同事模拟的colleague-skill，升级成通用性更强的dot-skill v2.0通用技能引擎，突破了原本只能用于职场同事的局限。首先适用的人物分成三大类，分别是同事导师这类职场角色、家人伴侣这类私人关系角色，还有名人创作者、虚拟小说人物这类公众角色；底层采用人格+工作能力双层架构，既能复刻一个人的说话性格，也能复刻他处理具体工作的做事方式。同时这个Skill兼容性拉满，能够直接部署运行在Claude Code、Hermes、OpenClaw、Codex四款主流Agent载体里面，Claude Code原生支持斜杠命令调用，Hermes可以一键快速安装启用，剩下两款也能做到完整兼容、按名称调用，整体通用性更强、适配的角色范围更广、可搭载的智能载体也更多。

## 第三张PPT（支持的数据来源）
很多人会好奇，我们到底拿什么资料去“蒸馏”一个人，这里给大家分两大类说明。第一种是自动采集模式，对接了飞书、钉钉、Slack这些主流办公软件，填入对应人员名字就能全自动抓取聊天消息、在线文档、表格内容，只有钉钉有一点限制，没法读取历史过往消息，Slack还需要管理员提前部署机器人，免费版有效期是90天。第二种是手动上传或者直接输入内容，我们可以导出微信聊天记录文件、上传PDF、截图、邮件文件，也能导入飞书导出数据包，或是直接粘贴Markdown文字内容。整体流程就是汇总各类原始数据，系统自动整理信息，分别提取这个人的性格人设和工作行事逻辑，最后合成专属Skill；原始素材越全面丰富，最终生成出来的AI就越贴合真人，不管是办公沟通记录，还是各类文档素材，全都能作为生成素材使用。

## 第四张PPT（安装与使用步骤）
我带大家实操看懂dot-skill怎么安装、怎么上手使用。安装方式非常简单，我们直接在对应的Agent里面发送安装指令附带项目地址，就能自动完成部署，四款不同载体对应的存放路径各不相同，Hermes还需要额外运行指定脚本完成安装，全部装好之后，我们在任意载体输入`/dot-skill`这个统一入口指令就能启动功能。完整使用步骤一共五步，启动主命令之后，先选定要创建的人物类型，填写人物姓名、基础背景、性格标签等信息，接着选定对应的资料数据源完成素材导入，系统自动生成专属人物Skill，后续我们直接用自定义命名指令就能随时调用这个角色。同时平台配套了完善管理命令，既能单独调用这个人的工作能力、或是单纯调取人格对话，也可以查看全部已创建技能、回退历史版本，整体上手门槛低，创建之后调用、管理都很便捷。

而 Codex 最新推出的 Record & Replay，最大的变化就在于——你不用写代码，也不用写复杂的 Prompt，只需要做一遍给 AI 看。

就像教一个新同事一样。

你第一次操作的时候，AI 会在旁边静静地观察：你打开了哪些软件、点击了哪些按钮、输入了哪些内容、为什么这么做。

等你做完之后，它不会只是保存一段录像，也不会只是记录鼠标点击的位置，而是真正理解整个工作流程，然后自动帮你生成一个可以反复使用的 Skill（技能）。

以后你再遇到同样的工作，就不用重新操作了，只需要一句话，例如：

“帮我上传今天的视频。”

或者：

“按照昨天的流程，再生成一次日报。”

Codex 就会自动完成整个过程。

所以，Record 可以理解为教 AI 一次；Replay 可以理解为让 AI 无限次帮你完成同样的工作。

这里最重要的一点是，它记录的不是鼠标轨迹，而是你的工作方法。即使以后换了一份文件、换了一张图片、甚至网页有一些变化，它依然能够理解你的目标，并继续完成任务。

所以，我们可以用一句最简单的话来总结这个功能：

以前，我们需要告诉 AI 应该怎么做；现在，我们只需要做一次给 AI 看。

也就是说：

Record 一次，沉淀一个 Skill；Replay 无数次，让重复工作真正实现自动化。

这也是 OpenAI 希望带来的新工作方式——从“写 Prompt”进化到“教 AI 工作”。未来，每个人都可以把自己的工作经验变成一个个可复用的 AI Skill，就像培养一位越来越熟悉你工作习惯的智能助手一样。
